# landing_copy

Before writing, read:

- `../../references/brand-system.md`
- `../../references/business-constraints.md`
- `../../references/tool-routing.md`
- `../../references/asset-map.md`

## Principle

Conversion first, aesthetics second.

The page must be brutally clear before it becomes decorative.

## Default website structure

1. hero
2. real value proposition
3. services in compact form
4. process or diagnostic logic
5. contact

## Rules

- no overexplaining
- no generic premium language
- no invented specialization
- make the contact path obvious
- if editing the actual site, align with the current ROSFIX visual system in code

## Output

Return the copy in deployable sections with labels.

If the user wants execution, inspect local files first and then use BrowserOS or Vercel only when verification is needed.
